## Context

任务弹窗的阶段计划已经维护 `TaskPhase[]`，每个阶段包含执行者、起止时间和进度。成员排期页也已经用同一份阶段数据按成员聚合排期。当前缺口是：用户在任务弹窗中为阶段选择执行者时，需要离开弹窗或凭记忆判断该成员近期负载。

## Goals / Non-Goals

**Goals:**
- 在任务弹窗当前选中的阶段行内提供执行者未来两周排期预览。
- 预览使用当前项目内的任务阶段数据，并包含当前表单中的草稿阶段。
- 保持预览轻量，不添加“张三 · 未来两周排期”之类标题行。
- 复用现有工作日、阶段时间和成员排期定位逻辑。

**Non-Goals:**
- 不新增任务、阶段或成员数据字段。
- 不改变任务保存 API 或成员排期页面功能。
- 不提供排期自动冲突阻止或自动推荐执行者。
- 不把完整成员排期页嵌入任务弹窗。

## Decisions

1. 预览作为阶段行的局部辅助信息。

   只在阶段被选中且已选择执行者时，在该阶段行下方展示日期轴和排期条。这样用户关注哪个阶段，就看到哪个成员的容量，也避免多个阶段同时展开挤占弹窗空间。备选方案是在弹窗顶部显示一个全局排期面板，但会脱离具体阶段上下文，也更占空间。

2. 时间窗口固定为“今天起未来两周”。

   窗口从本地今天 00:00 开始，到第 15 天 00:00 之前结束，即包含今天和之后 14 天。这个范围足够覆盖近期分配判断。预览网格使用紧凑槽位并按容器宽度等分，优先一行显示完整两周范围。备选方案是跟随阶段开始时间，但用户尚未填写阶段时间时会失去预览价值。

3. 数据来源合并已保存任务和当前表单草稿。

   已保存任务来自 `taskStore.tasks` 中同项目、同执行者、与窗口有交集的阶段。当前正在编辑的任务应排除已保存版本，再用表单里的阶段作为草稿数据补回，避免同一任务旧值和新值重复。当前阶段草稿用不同样式标识。

4. 预览只展示必要信息。

   预览显示日期轴、阶段条、任务标题和阶段名；更完整的迭代、时间范围、进度可放在 `title`/tooltip。标题行被明确移除，避免重复“选中成员是谁”的信息。无排期时显示一行轻量空状态。

5. 优先复用工作日时间槽工具。

   排期条定位使用 `generateWorkSlots` 和 `getWorkSlotGridRange` 的同一套语义，确保休息日、工作时间和跨休息日裁剪与时间轴/成员排期保持一致。V1 可默认折叠休息日，不在任务弹窗内增加折叠开关。

## Risks / Trade-offs

- [弹窗内容变高] -> 只展示当前选中阶段行的预览，并使用紧凑高度。
- [两周排期横向过宽] -> 使用按容器等分的紧凑时间槽，优先完整显示而不是依赖横向滚动。
- [草稿和已保存数据重复] -> 聚合时排除当前任务 ID，再合并表单阶段。
- [短阶段条标签放不下] -> 条内保留短标签，完整信息放在 tooltip。
- [用户误以为系统会阻止冲突] -> V1 只做可视提示，不改变保存校验规则。
