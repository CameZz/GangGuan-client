## Why

当前任务单只有一个固定任务阶段和一个负责人，无法表达“不同任务选择不同阶段、按任务实际流程排序、每个阶段由不同成员推进”的执行方式。项目需要把阶段库沉淀在项目内，并让任务阶段由参与者进度自动推进，减少手动改阶段带来的不一致。

## What Changes

- 在项目中维护项目级阶段库，阶段库可用于当前项目下的任务单。
- 创建任务单时从项目阶段库选择需要的阶段，并支持按任务实际流程排序。
- 每个任务阶段可指定执行者，任务单不要求经过项目阶段库中的全部阶段。
- 任务阶段不可直接手动修改，当前阶段由阶段进度自动计算。
- 参与者可在任务单内通过进度条记录自己负责阶段的完成度；普通用户只能改自己的进度，管理员可改所有人的进度。
- 阶段进度达到 100% 后自动完成并进入下一个未完成阶段；全部阶段完成后任务单自动完成。

## Capabilities

### New Capabilities
- `project-task-phase-library`: 项目级任务阶段库、任务阶段计划、阶段执行者、参与者进度记录和自动阶段流转。

### Modified Capabilities

## Impact

- 影响项目数据模型、任务数据模型、mock 数据、项目编辑/详情、任务创建/编辑弹窗、参与者列表、看板/列表阶段展示和任务状态/阶段流转逻辑。
