## Context

当前系统的 `Task` 只有一个固定 `stage` 字段、一个 `assigneeId` 和一组 `participants`。这适合简单任务，但不能表达“同一项目内维护阶段库，单个任务按实际情况选择部分阶段并排序，每个阶段由不同成员推进”的流程。

本设计建立项目级阶段库，并把任务单的真实阶段状态从单个 `stage` 字段迁移为阶段计划。`stage` 和 `assigneeId` 继续作为兼容展示字段，但其值应由当前任务阶段推导。

## Goals / Non-Goals

**Goals:**
- 在项目中保存阶段库，阶段库只服务当前项目。
- 创建任务单时从当前项目阶段库选择阶段、排序并指定执行者。
- 任务阶段不可通过普通字段直接修改，当前阶段由阶段进度自动计算。
- 参与者列表展示阶段进度条，普通用户只可修改自己的阶段进度，管理员可修改全部。
- 阶段进度达到 100% 后自动进入下一个未完成阶段，全部阶段完成后任务完成。
- 保持旧任务、看板、列表、成员排期和筛选的基本兼容。

**Non-Goals:**
- 不支持跨项目共享阶段库。
- 不支持需求单自身进入阶段执行流程。
- 不引入多级阶段、阶段依赖图或并行阶段阻塞规则。
- 不把阶段库修改强制同步到历史任务。

## Decisions

1. 在 `Project` 上新增阶段库字段。

   使用 `ProjectPhaseTemplate[]` 保存项目阶段库，每个模板包含 `id`、`name`、`order`、`enabled`。阶段库跟随项目数据保存，避免全局阶段污染不同项目流程。备选方案是全局阶段库，但会让不同项目的流程配置互相影响。

2. 在 `Task` 上新增阶段计划字段。

   使用 `TaskPhase[]` 保存任务实际选中的阶段，每个阶段包含 `id`、`templateId`、`name`、`order`、`assigneeId`、`progress`、`status`。任务阶段保存 `name` 快照，项目阶段库改名或停用不会静默改变历史任务。备选方案是只保存 `templateId`，但历史任务会随模板变化产生歧义。

3. `stage` 和 `assigneeId` 作为派生兼容字段保留。

   当前阶段等于按 `order` 排序后第一个 `progress < 100` 的阶段。`stage` 用当前阶段名或模板标识映射，`assigneeId` 用当前阶段执行者。这样现有看板、列表、筛选和任务卡片可以逐步迁移，而不是一次性重写全部视图。

4. 阶段流转由进度驱动。

   用户不直接修改任务当前阶段。参与者通过进度条更新自己负责阶段的 `progress`。当进度达到 100 时该阶段变为完成；系统重新计算当前阶段。所有阶段完成时任务状态变为 `done`。如果任一阶段进度大于 0 且任务未完成，任务状态可进入 `in-progress`。

5. 参与者列表展示阶段执行行。

   任务弹窗的参与者区域按任务阶段展示执行者和进度。普通用户只能编辑 `assigneeId` 等于自己的阶段进度，管理员可编辑任何阶段进度。若同一成员负责多个阶段，列表按阶段显示多行，避免合并后丢失阶段顺序和权限边界。

6. 旧任务通过归一化兼容。

   对没有 `phases` 的任务，读取时生成一个兼容阶段：名称来自旧 `stage`，执行者来自旧 `assigneeId`，进度根据任务状态推导。保存后逐步写入新的 `phases` 字段。需求单保持无执行阶段。

## Risks / Trade-offs

- [项目阶段库修改影响历史理解] -> 任务阶段保存名称快照，模板变更只影响后续任务。
- [自动流转和旧手动状态冲突] -> 任务阶段作为当前阶段来源，旧 `stage` 只做兼容显示。
- [同一成员多个阶段在参与者列表中重复] -> 按阶段显示，保证进度和权限清晰。
- [旧任务缺少阶段计划] -> 读取归一化生成兼容阶段，避免旧数据无法打开。
- [全部阶段完成后用户仍想返工] -> V1 可通过管理员把某阶段进度调低触发重新计算，不新增专门返工流程。

## Migration Plan

1. 为现有项目初始化阶段库，默认使用当前 `TASK_STAGES`。
2. 为现有任务补齐 `phases`，需求单保持空阶段。
3. 更新创建/编辑任务流程，使用项目阶段库生成任务阶段计划。
4. 将看板、列表、任务卡片中的阶段展示切换为派生当前阶段。
5. 更新参与者区域，加入阶段进度条和权限控制。

## Open Questions

- 项目阶段库入口放在项目详情页，还是新增项目设置入口？
- 阶段库模板是否需要支持颜色或说明字段？
- 管理员把已完成阶段进度调低时，是否需要追加历史记录提示“任务回退”？
