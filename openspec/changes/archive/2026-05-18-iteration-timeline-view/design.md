## Context

当前 `TimelineView.vue` 以角色为维度，按参与者显示甘特条。用户需要一个以**任务为行、日期为列**的网格视图，快速查看迭代内所有任务的时间安排。

现有数据结构：
- `Planning.createdAt` / `Planning.deadline` 定义迭代时间范围
- `Task.participants[].startTime` / `endTime` 定义每个参与者的时间区间
- 一个任务可有多个参与者，每个参与者有独立的时间段

## Goals / Non-Goals

**Goals:**
- 以网格形式展示迭代内所有任务的时间安排
- 每天划分为 4 个时间格子（上午/下午/晚上/深夜）
- 时间范围固定为选中迭代的 `createdAt` → `deadline`
- 任务横条根据参与者的时间区间在网格中定位
- 保留现有的筛选功能（按阶段、优先级、负责人）

**Non-Goals:**
- 不支持拖拽调整任务时间
- 不支持在网格中直接创建/编辑任务
- 不改变现有数据模型或 Store 结构

## Decisions

### 1. 时间轴布局：任务行 × 日期列网格

**选择**：CSS Grid 布局，左侧固定任务信息列，右侧为可横向滚动的时间网格。

**理由**：与现有 KanbanView/ListView 的布局风格一致，左侧 sticky 列保证任务信息始终可见。

**替代方案**：Canvas 绘制 — 性能更好但开发成本高，且不利于样式统一。

### 2. 每天 4 格的时间分段

**选择**：将每天分为 4 个等宽格子，分别代表 00:00-06:00、06:00-12:00、12:00-18:00、18:00-24:00。

**理由**：与用户描述的"每天分四个格子"一致，覆盖全天 24 小时。

### 3. 任务横条的定位逻辑

**选择**：根据 `participant.startTime` 和 `participant.endTime` 计算横条在网格中的起止列位置。一个任务的多个参与者横条垂直堆叠显示。

**理由**：复用现有数据结构，无需新增字段。参与者横条按角色着色，与现有甘特图保持视觉一致性。

### 4. 复用现有 TimelineView.vue

**选择**：重构现有 `TimelineView.vue`，替换为新的网格视图。

**理由**：路由中已配置 `/timeline` 指向该视图，重构比新建文件更简洁。

## Risks / Trade-offs

- **迭代无 deadline 时**：需提供默认值（如 createdAt + 14 天），避免网格宽度为 0 → 使用 fallback 逻辑处理
- **迭代时间跨度大时**：列数过多可能影响性能 → 限制最大显示天数或使用虚拟滚动（暂不实现，后续优化）
- **任务无参与者时间数据时**：横条不显示，仅显示任务行 → 符合预期，空行提示用户补充排期
