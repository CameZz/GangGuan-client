## ADDED Requirements

### Requirement: 自己参与的任务筛选
系统 SHALL 在看板和列表任务视图中提供“自己参与的任务”筛选，并在启用后仅显示当前登录用户参与的任务。

**目的**：帮助用户快速聚焦自己实际参与的任务，减少在项目和迭代任务集中查找的成本。

**输入**：当前登录用户、任务集合、任务 `participants` 数组、现有筛选条件。

**输出**：只包含符合所有筛选条件且当前用户出现在 `participants[].memberId` 中的任务集合。

**验证方法**：在看板和列表中启用/关闭该筛选，检查任务数量和任务内容是否符合参与者关系。

#### Scenario: 默认不限制参与关系
- **WHEN** 用户进入看板或列表且未启用“自己参与的任务”筛选
- **THEN** 系统按现有项目、迭代、状态、阶段、优先级、负责人和废弃任务规则显示任务

#### Scenario: 启用自己参与的任务筛选
- **WHEN** 当前用户 id 为 `user-1` 且启用“自己参与的任务”筛选
- **THEN** 看板和列表仅显示 `participants` 中存在 `memberId` 为 `user-1` 的任务

#### Scenario: 与其他筛选条件叠加
- **WHEN** 用户同时启用“自己参与的任务”筛选并选择状态、阶段、优先级或负责人筛选
- **THEN** 系统仅显示同时满足所有筛选条件的任务

#### Scenario: 当前用户不存在
- **WHEN** 没有当前登录用户且启用“自己参与的任务”筛选
- **THEN** 系统显示空任务结果且不抛出运行时错误
