## Why

当前建单只有“任务”一种类型，无法表达“一个需求需要拆成多个执行任务”的管理方式，导致需求目标和具体执行项混在一起。新增需求单与任务单的层级关系，可以让 PM 先记录需求，再拆分给不同角色执行。

## What Changes

- 建单时支持选择“需求单”或“任务单”。
- 需求单可包含多个任务单，用于承载需求说明、拆分进度和子任务管理。
- 需求单下面只能创建任务单，不允许再创建需求单或形成多级需求嵌套。
- 任务单可选择所属需求单，并继续保留负责人、参与者、排期、阶段、状态等执行字段。
- 成员排期、时间轴和工作量统计只计算任务单，不计算需求单。
- 需求单状态和进度由其下任务单汇总展示。

## Capabilities

### New Capabilities
- `requirement-task-breakdown`: 需求单与任务单的两层拆分关系、建单类型选择、子任务管理和展示规则。

### Modified Capabilities
- 无

## Impact

- 影响任务类型定义、mock 数据、任务创建/编辑弹窗、看板、列表、项目详情、时间轴和成员排期过滤逻辑。
- 复用现有 task store、任务弹窗和任务视图，不新增后端依赖。
